title: "Web GUI"
description: "Manage backgrounds, review generated videos, and adjust settings through a local Flask web interface."
---

The bot ships with an optional web interface built on **Flask**. It runs as a separate process from `main.py` and gives you a browser-based way to browse generated videos and manage your background library.

<Note>
  The GUI does **not** generate videos. Use it alongside `main.py`, not as a replacement. Start `main.py` separately to run the video creation pipeline.
</Note>

## Starting the GUI

<Steps>
  <Step title="Install dependencies">
    Make sure you have installed all project dependencies. Flask and tomlkit are required for the GUI.

    ```bash
    pip install -r requirements.txt
    ```
  </Step>

  <Step title="Start the server">
    Run the GUI from the project root directory:

    ```bash
    python GUI.py
    ```

    The server starts on port **4000** and automatically opens `http://localhost:4000` in your default browser. If the tab does not open, navigate there manually.

    ```
    Website opened in new tab. Refresh if it didn't load.
    ```
  </Step>

  <Step title="Browse your videos">
    The home page (`/`) lists all videos that have been generated by `main.py`. Each entry is served from the `results/` folder and can be downloaded directly from the browser.
  </Step>

  <Step title="Manage backgrounds">
    Navigate to `/backgrounds` to see all currently configured background video options. From here you can add new custom backgrounds or delete existing ones.
  </Step>
</Steps>

## Routes

| Route | Method | Description |
|---|---|---|
| `/` | `GET` | Home page — lists all generated videos from `results/` |
| `/backgrounds` | `GET` | Background management page |
| `/background/add` | `POST` | Add a new custom YouTube background |
| `/background/delete` | `POST` | Remove a background by key |
| `/settings` | `GET` / `POST` | View and edit `config.toml` through a web form |
| `/videos.json` | `GET` | Serves the raw video index from `video_creation/data/videos.json` |
| `/backgrounds.json` | `GET` | Serves the background list JSON from the `utils/` directory |
| `/results/<name>` | `GET` | Serves individual video files from the `results/` directory |

## Adding a Custom Background

The `/background/add` form accepts four fields:

```
POST /background/add
```

| Field | Description | Example |
|---|---|---|
| `youtube_uri` | Full YouTube URL of the background video | `https://www.youtube.com/watch?v=abc123` |
| `filename` | Local filename to save the video as (no `-` characters) | `subway_surfers.mp4` |
| `citation` | Channel or creator name used for the watermark and cache key | `Kiloo` |
| `position` | Vertical scroll position. Use `center` for static backgrounds | `center` |

<Warning>
  Avoid using `-` in the `filename` field. The bot uses `-` as a separator between the citation and filename when building the cache path (e.g. `Kiloo-subway_surfers.mp4`). A `-` inside the filename itself can cause lookup failures.
</Warning>

After submitting the form, the new background is added to `utils/backgrounds.json` and becomes available as an option in `config.toml` immediately.

## Settings Page

The `/settings` route renders a form that maps directly to all fields in your `config.toml`. Saving the form writes changes back to disk using **tomlkit**, which preserves comments and formatting in the file.

<Tip>
  Changes made through the settings page take effect on the next run of `main.py`. You do not need to restart the GUI server.
</Tip>
