## Overview

The project includes a `Dockerfile` that builds a self-contained image based on `python:3.10.14-slim`. The image installs FFmpeg and all Python dependencies, then runs `main.py` as the container's entry point.

Use the Docker setup if you want to avoid managing a local Python environment, or if you are running the bot on a server.

<Warning>
  Docker does not handle interactive terminal prompts well. You must pre-populate `config.toml` before running the container — the bot will not be able to prompt you for missing values inside a non-interactive container session.
</Warning>

---

## Dockerfile

The `Dockerfile` at the root of the repository:

```dockerfile Dockerfile
FROM python:3.10.14-slim

RUN apt update
RUN apt-get install -y ffmpeg
RUN apt install python3-pip -y

RUN mkdir /app
ADD . /app
WORKDIR /app
RUN pip install -r requirements.txt

CMD ["python3", "main.py"]
```

The image:
- Uses `python:3.10.14-slim` as its base
- Installs `ffmpeg` via `apt`
- Copies the entire repository into `/app`
- Installs all Python dependencies from `requirements.txt`
- Runs `python3 main.py` on container start

## Before you build

### Create config.toml

Because the container runs non-interactively, you must create and populate `config.toml` before building or running. The bot reads this file at startup and exits if required values are missing.

Copy the template and fill in your values:

```bash
cp utils/.config.template.toml config.toml
```

At minimum, set your Reddit credentials and thread settings:

```toml config.toml
[reddit.creds]
client_id = "your_client_id"
client_secret = "your_client_secret"
username = "your_reddit_username"
password = "your_reddit_password"
2fa = false

[reddit.thread]
subreddit = "AskReddit"
random = true
max_comment_length = 500
min_comment_length = 1
min_comments = 20

[settings]
theme = "dark"
times_to_run = 1
allow_nsfw = false
opacity = 0.9
resolution_w = 1080
resolution_h = 1920

[settings.tts]
voice_choice = "googletranslate"
random_voice = false
no_emojis = false
silence_duration = 0.3

[settings.background]
background_video = "minecraft"
background_audio = "lofi"
background_audio_volume = 0.15
```

<Tip>
  Use `voice_choice = "googletranslate"` or `"pyttsx"` when running in Docker to avoid needing additional API keys or session tokens for your first test.
</Tip>

## Build the image

From the root of the repository:

```bash
docker build -t rvmt .
```

This tags the image as `rvmt` (Reddit Video Maker Tool). The build step installs all dependencies and may take a few minutes on the first run.

---

## Run the container

```bash
docker run -v $(pwd)/assets:/app/assets -v $(pwd)/config.toml:/app/config.toml rvmt
```

The `run.sh` script at the repository root provides a reference command:

```sh run.sh
docker run -v $(pwd)/out/:/app/assets -v $(pwd)/.env:/app/.env -it rvmt
```

Mount points to be aware of:

| Host path | Container path | Purpose |
|-----------|---------------|---------|
| `./assets` | `/app/assets` | Output videos and downloaded backgrounds |
| `./config.toml` | `/app/config.toml` | Bot configuration (required) |

<Note>
  Output videos are written into the `assets/` directory inside the container. Mount a host directory to `/app/assets` so the files are accessible after the container exits.
</Note>

## Troubleshooting

**The bot exits immediately asking for configuration**

The container cannot display interactive prompts. Make sure `config.toml` exists and all required fields are populated before running. See the [configuration reference](/reference/config-toml) for a full list of required fields.

**Playwright fails inside the container**

The base image is `python:3.10.14-slim` and does not include all browser dependencies by default. If you need Playwright inside Docker, add the following to your `Dockerfile` before the `pip install` step:

```dockerfile
RUN pip install playwright && python3 -m playwright install --with-deps chromium
```

**FFmpeg not found**

The `Dockerfile` installs `ffmpeg` via `apt`. If you are using a custom base image, make sure `ffmpeg` is available on the `PATH`.
