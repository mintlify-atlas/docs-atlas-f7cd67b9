Background assets are downloaded automatically from YouTube on first use and cached locally. Subsequent runs use the cached files and do not require a network connection.

<Note>
  Background files are downloaded on first use. The initial run may take longer depending on your internet connection. Downloaded files are stored in the `assets/backgrounds/` directory.
</Note>

## Using a Random Background

Set either `background_video` or `background_audio` to an empty string to have the bot pick a random option from the available pool on each run:

```toml
[settings.background]
background_video = ""
background_audio = ""
```

<Tip>
  Using an empty string is a convenient way to add variety to your videos without manually changing config values between runs.
</Tip>

## Disabling Background Audio

To generate videos without background music, set the volume to `0` rather than removing the key:

```toml
[settings.background]
background_audio_volume = 0
```

## Adding Custom Backgrounds

You can extend `utils/background_videos.json` and `utils/background_audios.json` with your own entries. Each video entry follows the format:

```json
"your-key": [
  "https://www.youtube.com/watch?v=VIDEO_ID",
  "output-filename.mp4",
  "Credit Name",
  "center"
]
```

Each audio entry follows the format:

```json
"your-key": [
  "https://www.youtube.com/watch?v=VIDEO_ID",
  "output-filename.mp3",
  "Credit Name"
]
```

After adding an entry, use your new key as the `background_video` or `background_audio` value in `config.toml`.
